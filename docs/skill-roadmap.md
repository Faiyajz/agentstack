# AgentStack Skill Roadmap

AgentStack should stay broad enough for real software careers, but focused enough that each skill has a clear trigger and finish line.

## Core Delivery

- `product-brief`: turn an idea into users, goals, scope, risks, and success criteria.
- `design-doc`: make architecture decisions reviewable before implementation.
- `spec`: write implementation-ready technical design.
- `plan`: split work into agent-sized tasks.
- `implement` and `tdd`: execute scoped changes with verification.
- `review`: review correctness, security, simplicity, and tests.
- `commit` and `branch`: keep Git history traceable.

## Engineering Domains

- `fullstack-feature`: ship one vertical product slice across UI, API, data, tests, and rollout.
- `backend-api`: build API, database, queue, and service changes with contracts and failure behavior.
- `frontend-ui`: build accessible, responsive, browser-verified interfaces.
- `devops-deploy`: prepare CI/CD, deployment, configuration, rollback, and smoke checks.
- `server-admin`: inspect and operate Linux servers without destructive surprises.
- `security-review`: evaluate practical application, infrastructure, and workflow risk.
- `incident-response`: triage production problems, mitigate, communicate, and capture follow-up work.

## Career and Management

- `interview-prep`: coordinate role-specific practice, questions, feedback, and improvement plans.
- `coding-interview`: algorithms, debugging, live coding, and communication practice.
- `system-design-interview`: architecture, scale, data, reliability, and tradeoff practice.
- `behavioral-interview`: STAR stories, leadership, conflict, ownership, and project narratives.
- `technical-article`: researched technical writing with verified examples and optional author voice.
- `dsa-teacher`: beginner-friendly DSA and competitive programming explanations, practice sets, and rated problem paths.
- Future: `engineering-manager`, `project-manager`, `resume-review`.

## Suggested Future Skills

- `database-design`: schema, indexing, migrations, transactions, and data retention.
- `api-contract`: OpenAPI, versioning, compatibility, idempotency, and error shapes.
- `observability`: logs, metrics, traces, dashboards, alerts, and SLOs.
- `performance`: profiling, load testing, bottleneck analysis, and regression checks.
- `cloud-architecture`: AWS/GCP/Azure architecture decisions, cost, security, and reliability.
- `threat-model`: structured security modeling before implementation.
- `pentest-triage`: validate and prioritize vulnerability findings.
- `docs-writer`: write user docs, runbooks, ADRs, and release notes.
- `product-manager`: roadmap shaping, requirements, tradeoffs, and launch planning.
- `startup-mvp`: choose a small marketable version, scope it, and ship it.
