# DSA Patterns

Use these buckets to classify problems.

## Core Beginner Patterns

- implementation and simulation
- math and number theory basics
- frequency counting
- sorting
- prefix sums
- two pointers
- sliding window
- binary search
- greedy
- stack
- queue and deque
- hashing
- basic recursion

## Intermediate Patterns

- binary search on answer
- intervals
- monotonic stack
- heap and priority queue
- linked list
- tree DFS/BFS
- graph DFS/BFS
- shortest path basics
- union find
- backtracking
- dynamic programming basics
- bit manipulation
- string matching basics

## Competitive Programming Patterns

- constructive algorithms
- casework
- invariants
- parity
- modular arithmetic
- greedy with sorting
- prefix/suffix contribution
- mex
- frequency and counting tricks
- interactive reasoning, only when requested

When a problem has multiple patterns, choose:

1. the pattern that unlocks the solution
2. the pattern the learner should remember
3. secondary patterns as notes
