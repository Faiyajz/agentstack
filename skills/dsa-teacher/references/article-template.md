# One Problem Article Template

Use this for each problem article.

```markdown
# <Problem Title> - Step by Step Explanation

## Who This Is For

One short sentence describing the learner and prerequisite.

## Problem In Simple Words

Explain the task without copying the original statement.

## Input and Output Meaning

Explain what each value represents.

## Tiny Example

Use the smallest useful example and solve it by hand.

## First Thought: Brute Force

Explain the most obvious solution.

## Why Brute Force Fails

Mention time complexity or repeated work when relevant.

## Key Observation

State the one idea that unlocks the problem.

## Better Solution

Explain the algorithm in plain steps.

## Dry Run

Show table or numbered state changes.

## Code

Use the requested language.

## Code Explanation

Explain important lines or blocks.

## Complexity

- Time:
- Space:

## Edge Cases

List edge cases and why the solution handles them.

## Similar Questions

List related problems and what they teach.

## Final Intuition

One paragraph that makes the pattern memorable.
```

Keep the article focused on one question. Put practice sets in a separate overview article when needed.
