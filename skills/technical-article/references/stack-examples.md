# Stack Examples

Choose the smallest complete project that proves the article.

## Node.js

- Use package manager conventions already present or requested.
- Include `package.json`, source, tests or a runnable check.
- Verify with install, test, and run commands.

## Python

- Include `pyproject.toml` or `requirements.txt`.
- Prefer a virtual environment or documented tool workflow.
- Verify imports, tests, and command execution.

## Frontend

- Use the existing framework when inside a repo.
- Include accessible UI states and browser verification when relevant.
- Verify build, tests, and rendered interaction.

## Backend/API

- Include API routes, validation, error behavior, and tests.
- Use local storage or SQLite when practical to avoid external services.
- Document request examples.

## PHP/Laravel

- Use a complete Laravel project for Laravel-specific articles.
- Prefer SQLite for examples unless the article requires another database.
- Include feature tests for the main behavior.

## DevOps/Docker

- Include `Dockerfile`, `docker-compose.yml`, config, and smoke checks.
- Avoid requiring cloud credentials for the default path.
- Document cleanup commands.

## Database

- Include schema, seed data, queries, and expected output.
- Explain indexes and query plans when performance is central.

## Architecture

- Use Mermaid or clear diagrams.
- Include assumptions, failure modes, and tradeoffs.
- Add a small config/template example if code would be artificial.

## Project Management

- Include templates and filled examples.
- Show how the template changes decisions or execution.
